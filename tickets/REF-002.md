# REF-002 - Bybit Instrument Event Source Feasibility Audit

**Priority:** P0
**Status:** ACCEPTED - NO AUTHORITY
**Recommendation:** NO_AUTHORITY (REVIEW-0102)
**Dependencies:** REF-001, BYB-001, AUD-003, RES-001 (accepted)
**Layer:** research evidence / point-in-time reference events
**Architecture:** no ADR, schema, migration, or production implementation authorized

## Objective

Determine whether official Bybit instrument metadata, trade archives, and announcements can support
historical listing/delivery/delisting events or only prospective bitemporal snapshots.

## Bounded Scope

- BTCUSDT linear perpetual listing metadata plus the accepted 2020-03-25 trade archive edge.
- BTCUSDU26 inverse future as a scheduled future-delivery exemplar; do not call it already delivered.
- Exactly one actually settled inverse instrument selected deterministically from an official Bybit
  `status=Settled` response, with instrument metadata, archive edge, and announcement attempt.

## Required Decision

Publish exactly one recommendation:

1. `HISTORICAL_EVENT_AUTHORITY`;
2. `PROSPECTIVE_ONLY_AUTHORITY`; or
3. `NO_AUTHORITY`.

Any unknown historical known-time, revision, state-transition, announcement, mapping, licensing, or
raw-lineage gate prevents `HISTORICAL_EVENT_AUTHORITY`.

## Out Of Scope

- Reference or source implementation, schemas, migrations, ADRs, and generated datasets.
- Universe construction, FX, fees, funding, routes, factors, portfolios, or live work.
- Treating first/last trade as exact listing/delisting without a confidence classification.

## Stop Condition

Publish evidence records, set REF-002 to `AWAITING_REVIEW`, return control to Reviewer, retain
`Next ticket authorized: NONE`, and stop.
