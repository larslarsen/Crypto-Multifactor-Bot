# AUD-005 — Provider-candle comparison by explicit comparable dimensions

**Priority:** P2
**Status:** DRAFT
**Dependencies:** AUD-002 (source_audit toolkit, ACCEPTED at `899fb7c802dc4ba9b951118598417aef6d22cdcb`)
**Layer:** source-audit / bar-reconstruction

## Objective

Allow `source_audit.compare_bars` to compare reconstructed bars against provider
candles by explicit comparable dimensions when optional fields (e.g.
`trade_count`) are absent, instead of raising a hard error.

## Background

The Sprint 003 bounded trade-to-bar check (`bar_reconstruction_comparison.json`)
reconstructed 10 one-minute bars from 5437 Binance aggTrades successfully, but the
toolkit `compare_bars` could not complete because Binance klines carry no
`trade_count` field. The toolkit hard-requires `trade_count` on mapping-based
provider bars. The reconstruction result was preserved (status `partial`) and the
aggTrades record count was explicitly flagged as NOT equivalent to a kline's
raw-trade count.

## Deliverables

- Accept an explicit set of comparable dimensions (open/high/low/close/base volume/
  quote volume) and tolerate absence of optional fields via caller-supplied
  tolerance (e.g. `trade_count_tolerance` may also mean "skip when absent").
- When `trade_count` is absent, compare the available dimensions and report the
  trade-count dimension as `not_comparable` rather than failing the whole check.
- Add focused tests with klines that omit `trade_count`.
- Never present an aggregate record count as a raw-trade count.

## Non-blocking

Does **not** block RAW-001. RAW-001 implements the raw-object writer; this ticket
only improves an audit-tool bar comparison.
