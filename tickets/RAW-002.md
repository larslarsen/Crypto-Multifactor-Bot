# RAW-002 — Harden publication-receipt verification against symlink substitution

**Priority:** P3  \
**Dependencies:** RAW-001 (`ACCEPTED`)  \
**Layer:** ingest + catalog
**Status:** DRAFT

## Objective

Harden `verify_publication_receipt` so a published object whose canonical path is
replaced by a symlink after safe publication, but before catalog verification, is
rejected rather than silently accepted.

## Scope

- Construct the expected canonical path **lexically** without resolving the final
  component.
- Inspect every path component with `lstat` (do not follow symlinks).
- Reject a symlinked canonical final path **before** following it.
- Reject paths resolving outside the configured store root.
- Add a regression test where the canonical object path is replaced by a symlink after
  publication but before catalog verification.

## Explicit non-goals

- This is defense-in-depth against external post-publication filesystem mutation.
- It does **not** block acceptance of RAW-001 (the core writer and catalog invariants
  hold; exploitation requires external tampering).
- It is **not** authorized for implementation now.

## Agent stop condition

Backlog only. Do not begin implementation until explicitly promoted to an active ticket.
