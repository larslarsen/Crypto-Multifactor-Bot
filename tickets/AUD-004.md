# AUD-004 — Native headerless support for the Binance archive precision comparator

**Priority:** P2
**Status:** BLOCKED
**Dependencies:** AUD-002 (source_audit toolkit, ACCEPTED at `899fb7c802dc4ba9b951118598417aef6d22cdcb`)
**Layer:** source-audit / archive-precision

## Objective

Add native headerless support to `source_audit.compare_binance_archive_precision`
so the designed adjacent-archive precision-transition check runs on real Binance
daily dumps (aggTrades / klines), which are headerless CSVs.

## Background

The Sprint 003 audit (`research/sprint_003/12_AUDIT_EXECUTION.md`) found that
`compare_binance_archive_precision` hard-requires `has_header=True`, but Binance
daily dumps are headerless. The toolkit could not run the precision comparison on
real data. A runner-level adapter (`binance_precision_comparison_adapter.json`)
supplied equivalent evidence via `infer_timestamp_unit` and confirmed an
ms→us timestamp-unit transition, but the native comparator remains unusable on
daily dumps.

## Deliverables

- Support headerless archives by inferring the timestamp-column schema from the
  caller-supplied `timestamp_column` index rather than a header row.
- Preserve the existing explicit minimum-evidence and quality thresholds.
- Add focused tests covering headerless aggTrades/klines precision pairs.
- Document the headerless invocation path.

## Non-blocking

Does **not** block RAW-001. RAW-001 implements the raw-object writer; this ticket
only improves an audit-tool precision check.
