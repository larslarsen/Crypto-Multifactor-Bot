# FX-001 - Point-in-Time Stablecoin FX Readiness

**Priority:** P0
**Status:** AWAITING_REVIEW
**Dependencies:** CAT-001/001A, RAW-001/002, MAN-001, REF-001, BAR-001, AUD-003 (all accepted)
**Layer:** data platform / market FX
**Architecture:** readiness must determine whether an ADR is required before implementation

## Objective

Define a reviewable implementation contract for point-in-time stablecoin quote-to-USD conversion so
historical price, volume, universe, and return calculations never assume USDT, USDC, or another
stablecoin equals exactly one USD.

## Readiness Questions

- Which repository-audited source can provide historically reconstructable USD FX observations with
  observation time, availability time, revisions, and immutable raw lineage?
- What identifies an FX observation, stablecoin asset, venue/instrument source, interval, dataset,
  and policy version?
- How are depegs, missing observations, stale observations, multiple sources, confidence, and
  quarantine represented without forward-looking fills?
- Can existing reference, canonical-bar, manifest, and dataset-publisher contracts support the
  product, or is a new ADR, migration, or public schema required?
- What deterministic API and report allow downstream quote-volume conversion while preserving the
  original venue quote values and every FX component?

## Current Scope

Records and analysis only under `docs/reviews/FX-001_JR_READINESS_TASK.md`. Production source,
tests, migrations, generated datasets, provider collection, and research conclusions are not yet
authorized.

## Stop Condition

Publish the readiness report and return control to Reviewer with `Next ticket authorized: NONE`.
Do not begin implementation or another ticket.
