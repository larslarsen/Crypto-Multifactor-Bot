## Objective

The TSMOM winner (`tsmom_14_3`, +16.70%) was selected from a grid of 14 configurations (EXP-004). Holdout OOS (EXP-005) showed all train winners lost on test — `oos_supports_live_path: false`. However, full-window screen (EXP-007) showed `tsmom_14_3` as best, confirmed on PASS bars (PAPER-009). 

This ticket applies a **formal data-snooping / multiple-testing correction** to determine whether the winner survives statistical adjustment. If it does, that removes the selection-path policy blocker. If not, the candidate is a likely false discovery and must be rejected.

## Scope

1. **Multiple-testing correction** on the 14-config grid returns (full window, PASS daily data):
   - Bonferroni-adjusted p-value threshold or minimum detectable return
   - FDR (Benjamini-Hochberg) q-value ranking
   - **Required:** at least one method that accounts for the correlation structure of the returns (e.g., White's Reality Check, Hansen's SPA, or a bootstrap-based procedure)
2. **Decision rule:**
   - If `tsmom_14_3` return survives the most stringent correction → **multiple-testing risk resolved**, path to LIVE eligible
   - If it does not survive → **candidate is a false discovery**, archived; new research direction needed
3. **Artifact** `research/sprint_004/28_MULTIPLE_TESTING_ANALYSIS.json`:
   - all 14 config returns (full window on PASS bars)
   - correction method(s) used and parameters
   - whether winner survives
   - conclusion: `survives_correction: true/false`

4. **Do not** change lookback/skip, risk limits, bars, or promotion records.
