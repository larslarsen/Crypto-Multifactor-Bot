# FEE-001 - Point-in-Time Fee Schedules and Conservative Assumptions

**Priority:** P0
**Status:** ACCEPTED - REVIEW-0105
**Dependencies:** CAT-001/CAT-001A and REF-001 (accepted)
**Layer:** reference data / fee schedules
**Architecture:** implements existing Phase 3 contracts; no ADR required

## Objective

Add an exact, bitemporal reference-store contract for instrument fee schedules so later research can
distinguish official schedules from explicit conservative assumptions without inventing historical
fees or silently treating missing fees as zero.

## Required Contract

- Immutable `FeeSchedule` records scoped by instrument and canonical non-empty fee-tier ID.
- `maker_fee_rate` and `taker_fee_rate` stored and returned as exact `Decimal` dimensionless rates.
- Evidence class is exactly `OFFICIAL_SCHEDULE` or `ASSUMED_CONSERVATIVE`.
- Every record carries `BiTemporalWindow`, evidence metadata, and optional supersession lineage.
- No built-in provider, tier, rate, or zero-fee default.
- Point-in-time lookup requires both decision time and knowledge time.
- Missing schedules return no schedule; no venue, tier, current-state, or zero-cost fallback is allowed.

## Integrity Rules

- Fee rates must be finite `Decimal` values in `[0, 1)`; float inputs, NaN, infinities, negative rates,
  and rates at or above one are rejected. Negative maker rebates are outside this conservative v1.
- SQLite persists canonical decimal strings, never binary floating-point values.
- Instrument existence is validated inside the same atomic write unit.
- Bitemporal overlap for the same instrument/tier is rejected inside the atomic write unit.
- Supersession closes the prior knowledge window and inserts its replacement atomically with contiguous
  `known_from`; historical as-of queries continue to return the prior record.
- IDs are deterministic from immutable canonical identity material.

## Deliverables

- Forward-only migration `0007` for fee schedules.
- Typed reference models and public exports.
- Atomic write, supersession, and as-of query methods on `ReferenceStore`.
- Focused tests and integration evidence in a later Jr pass.

## Out Of Scope

- Numeric baseline assumptions or provider fee ingestion.
- Spread, impact, funding, borrow, FX conversion, routing, labels, portfolio simulation, or live work.
- CLI commands, network access, generated datasets, and changes to existing reference semantics.
- Closing architecture risk D-06; this ticket provides only the required substrate.

## Acceptance

- Fresh and previously migrated databases apply migration `0007` deterministically.
- Decimal rates round-trip exactly.
- Half-open valid/known boundaries, overlap rejection, atomic rollback, missing lookup, cross-tier
  independence, and contiguous supersession are covered by tests.
- Historical knowledge-time queries remain stable after correction.
- Full formatting, lint, type, layer, targeted-test, full-suite, and repository-control commands pass.
