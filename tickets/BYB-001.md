# BYB-001 - Bybit Public Trade-Archive Normalizer

**Priority:** P0
**Dependencies:** RAW-001, MAN-001, REF-001, AUD-003, BAR-001
**Layer:** ingest/market
**Status:** AWAITING_REVIEW
**Provider role:** `CONDITIONAL - CROSSCHECK`

## Objective

Normalize explicitly registered local Bybit perpetual-trade `.csv.gz` archives into a
source-specific typed trade dataset with immutable RAW/MAN lineage. This is the first
second-venue data adapter; it does not yet promote Bybit to primary, aggregate trades into
bars, or change BAR-001.

## Accepted evidence

The contract is grounded in Sprint 003's accepted audit:

- `research/sprint_003/sources/bybit.md`;
- `research/sprint_003/audit_results/bybit_archive_inspection.json`;
- `research/sprint_003/13_RESEARCH_LEAD_DECISIONS.md`.

Observed archives use this exact header:

```text
timestamp,symbol,side,size,price,tickDirection,trdMatchID,grossValue,homeNotional,foreignNotional
```

`timestamp` is Unix seconds with a fractional component carrying microsecond precision.
Linear and inverse contracts have different physical quantity meanings.

## Public source contract

Add `src/cryptofactors/ingest/bybit.py` with:

```text
normalize_bybit_trades(
    raw_objects,
    *,
    contract_style,
    source_symbol,
    venue_id,
    instrument_id,
    output_dir,
    code_commit,
    config_sha256=None,
    max_decompressed_bytes=209715200,
    max_rows=5000000,
) -> BybitTradeNormalizeResult
```

Required constants:

- dataset type/schema name: `bybit_trade_source`;
- schema version: `1`;
- transform name: `bybit_trade_normalizer`;
- transform version: `1`.

`contract_style` accepts only `linear` or `inverse`:

- `linear` -> canonical source market type `usdm`, `size_unit=base_asset`, schema variant
  `linear_base_size`;
- `inverse` -> canonical source market type `coinm`, `size_unit=contracts`, schema variant
  `inverse_contract_size`.

One invocation covers one resolved instrument and exact source symbol. Instrument identity is
caller-supplied from REF-001 and is never inferred from filenames or ticker text.

## Input requirements

- At least one immutable `RawObject` with `source_id == "bybit"`.
- Unique, safe `raw_object_id` values and regular non-symlink local files.
- Gzip and CSV are read as bounded streams; no whole-archive decompression in memory.
- Safety limits are positive, deterministic, identity-bearing configuration.
- The normalizer performs no discovery, HTTP, authentication, or network access.
- Archive filenames and directory names carry no identity or timestamp authority.

## Typed trade fields

Each accepted row preserves:

- caller-supplied `venue_id`, `instrument_id`, canonical source `market_type`,
  `contract_style`, schema variant, and physical `size_unit`;
- `event_time` as signed UTC microseconds;
- `source_timestamp` as the exact input token and
  `source_timestamp_unit="decimal_seconds"`;
- `source_symbol` and normalized `side` (`buy`/`sell`);
- `size`, `price`, `gross_value`, `home_notional`, and `foreign_notional` as finite
  `Decimal` values (never binary float);
- `tick_direction` and `trade_match_id`;
- `raw_object_id` and one-based source data-row number.

Do not manufacture cross-venue quantity equivalence:

- linear `size` may be identified as base-asset quantity;
- inverse `size` remains contract quantity;
- raw notional fields are preserved under their source meanings;
- inverse base/quote volume is not synthesized without an audited contract multiplier/spec.

## Timestamp contract

- Parse the source token with `Decimal`, never `float`.
- Multiplication by `1_000_000` must produce an integer; sub-microsecond values fail closed.
- Preserve trailing/short fractional forms exactly while mapping them to exact UTC us
  (`1585180700.0647` -> `1585180700064700`).
- Ascending, descending, and mixed archive order preserve original source row order.
- Record observed ordering per object. Direction reversals emit
  `bybit_trade_nonmonotonic_timestamp` as a warning, not a rejection: Sprint 003 did not
  establish global sort monotonicity, and row order is not economic identity.
- Do not silently sort source output; future aggregation must order deterministically by
  event time and trade ID.
- Coverage uses observed minimum/maximum event time, independent of source row order.

## Validation and quality contract

- Header names and order must match all ten audited fields exactly.
- Every data row must contain exactly ten columns.
- Symbol must exactly match the explicit `source_symbol`.
- Side must be `Buy` or `Sell`; tick direction must be one of `PlusTick`, `MinusTick`,
  `ZeroPlusTick`, or `ZeroMinusTick`.
- Size and price must be strictly positive; source notional fields must be finite and
  nonnegative; trade ID must be nonempty.
- Empty/header-only, bad gzip, malformed UTF-8/CSV, limit exceedance, invalid timestamps,
  mixed symbols, and physical-field failures are visible typed errors.
- Duplicate `trdMatchID` values are checked within and across raw objects. Preserve rows and
  emit distinct exact-duplicate versus conflicting-duplicate errors; never silently dedupe.
- Every row-level issue includes raw object ID and source row number. Cross-object duplicate
  issues include both raw object IDs.
- Any error makes the source dataset `REJECTED`; warnings produce `PASS_WITH_WARNINGS`;
  otherwise `PASS`.

Stable issue codes must cover at least:

- `bybit_archive_bad_gzip`;
- `bybit_archive_header_mismatch`;
- `bybit_archive_limit_exceeded`;
- `bybit_trade_row_width`;
- `bybit_trade_parse_failure`;
- `bybit_trade_symbol_mismatch`;
- `bybit_trade_invalid_timestamp`;
- `bybit_trade_nonmonotonic_timestamp`;
- `bybit_trade_duplicate_id`;
- `bybit_trade_conflicting_duplicate_id`;
- `bybit_trade_empty_observations`.

## Output and MAN-001 contract

Stage one `trades.parquet` and one `quality.parquet` per raw object. Logical output paths are
safe and deterministic under:

```text
bybit/{market_type}/trades/raw_{raw_object_id}/
```

Trade output specs include full partition evidence:

- raw object, venue, instrument, source symbol;
- canonical market type and contract style;
- schema variant and physical size unit;
- source timestamp unit and UTC-microsecond storage.

Return a MAN-001-publishable `PublishPlan` with:

- RAW_OBJECT dependency per input, role `bybit_trade_archive`;
- deterministic schema fingerprint and config identity;
- immutable code identity supplied by the caller;
- row/byte statistics and min/max event coverage;
- verified row counters under `REQUIRE_VERIFIER`;
- complete quality summary including contract/quantity/timestamp semantics and safety limits.

The complete plan must pass `verify_outputs` and a temporary catalog-registered
`DatasetPublisher.publish` integration test.

## Required Jr integration cases

- audited-shape linear and inverse rows;
- exact timestamps with six, fewer, and invalid excess fractional digits;
- descending and mixed-order archives preserve source order, report ordering diagnostics,
  and compute correct min/max coverage;
- strict header/width/symbol/side/tick/number validation;
- linear base-size versus inverse contract-size semantics;
- bounded gzip/row limits and bad gzip;
- empty/header-only input;
- duplicate/conflicting trade IDs within and across objects, with no hidden row deletion;
- source row/raw-object lineage and deterministic config/schema identities;
- safe paths, output verification, and full MAN-001 catalog publication;
- explicit no-network behavior.

## Out of scope

- REST acquisition, cursor pagination, raw REST-page capture, funding, mark/index data, and
  instrument metadata snapshots;
- trade-to-bar aggregation, native candle comparison, BAR-001 changes, or cross-venue
  consolidation;
- spot, options, delivered futures, fees, execution routes, and research factors;
- provider promotion beyond `CONDITIONAL - CROSSCHECK`;
- raw Bybit observations committed to Git.
