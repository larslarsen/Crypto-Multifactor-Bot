# LEG-001 — Register legacy local files without accepting their claims

**Priority:** P0  
**Dependencies:** RAW-001  
**Layer:** ingest  
**Status:** ACCEPTED

## Objective

Create a complete content/hash census of legacy local observations and artifacts without treating them as canonical or validated.

## Deliverables

- bounded-memory recursive scanner;
- file type, size, mtime, hash, relative path, and evidence class;
- configurable exclusions for secrets, environments, caches, and Git;
- no parsing required for registration;
- duplicates reported by content hash;
- generated inventory dataset and summary.

## Invariants

Registration does not imply acceptance. Original bytes are never rewritten. Raw data remains outside Git.

## Sr acceptance-fix drop (v1.2.1) — integrate first

Replace production file only:

- `src/cryptofactors/ingest/legacy_local.py` (scanner version **1.2.1**)

Source path in this change set: same relative path. Do not modify other production modules unless a test import requires an export already present.

Preserved invariants: source immutability; binary identity; descriptor-relative `O_NOFOLLOW` / `dir_fd` traversal; bounded external merge; exclusive staged publication.

## Acceptance blockers (must be covered by tests)

1. **Bounded directory iteration and streaming duplicates**  
   Directory entries must not be fully materialized via `os.listdir` into an in-memory name list for traversal. Duplicate-report generation must stream paths (no full per-hash path list materialization). Tests: census completes on a directory with many entries; a repeated content hash appears in the duplicate report with all relative paths.

2. **Reversible, collision-free path representation + deterministic order**  
   Raw name `b"\x80"` and a distinct UTF-8 file literally named `b64:gA==` must emit **two different** `relative_path` values. Inventory ordering must be stable across two scans of the same tree (compare inventory bytes ignoring `scanned_at_utc` in the summary only).

3. **Zero-byte regular file size**  
   An empty regular file must appear with `byte_size == 0` (not `null` / omitted).

4. **Output/work subtree exclusion before enqueue or hash**  
   With layout `legacy_root/out/preexisting.txt` and `output_dir=legacy_root/out`, `preexisting.txt` must **not** appear in the inventory. Summary `excluded_by_rule` must record output self-exclusion.
