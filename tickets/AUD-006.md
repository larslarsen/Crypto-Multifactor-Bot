# AUD-006 — Execution Risk & Live Authority Audit

**Priority:** P0
**Status:** READY
**Dependencies:** EXEC-001 (Paper Execution Runtime), PROMO-001 (Promotion Registry)
**Layer:** execution / governance
**Architecture:** Implements the mandatory risk-board authorization step required before Sequence #26 (Live Execution Routing).

## Objective

Conduct a comprehensive risk audit of the execution domain and promotion gates before granting authority to construct live order routing logic. This audit ensures that the software cannot accidentally leak paper/research states into live exchanges and that hard limits (leverage, single-asset caps, kill switches) are verifiable.

## Scope

- **State Separation Audit:** Verify that `PaperBroker` and the forthcoming live broker share no state and cannot cross-pollinate configurations.
- **Promotion Gate Verification:** Verify that `PROMO-001` hard-gates `LIVE_APPROVED` correctly (requires owner authority, evidence reference, and prospective paper observation via `HOLDOUT-001`).
- **Risk Limits:** Ensure the control plane enforces maximum gross leverage <= 1.0 and single asset max weight <= 0.15 before any live API keys can be loaded.
- **Kill-Switch Verification:** Verify that a documented kill-switch protocol exists and can instantly halt execution and flatten positions.

## Required Contract

- A written risk audit report (`docs/reviews/AUD-006_RISK_REPORT.md`).
- If deficiencies are found, they must be converted into blocking `FX-` tickets.
- If passed, the Lead Quant/Owner may explicitly authorize Sequence #26 (`EXEC-002` - Live Execution Routing).

## Acceptance (Jr)

1. Audit report committed to `docs/reviews/`.
2. Any necessary `FX-` tickets drafted and tracked.
3. `python3 scripts/check_repo_control.py`

## Phased Ownership

- Sr Dev (Strong Model): conduct the code/system audit and write the report.
- Jr Dev (Weak Model): records, Git, commit, push.

## Stop Condition

After Sr Dev implementation: Set state to AWAITING_REVIEW, wait for Lead Quant review, Next ticket authorized: NONE.
