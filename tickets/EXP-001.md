# EXP-001 — Experiment Bundles & Fingerprints

**Priority:** P0
**Status:** IN_PROGRESS
**Dependencies:** ASOF-001, SPLIT-001, LABEL-001 (all accepted)
**Layer:** validation / experiments
**Architecture:** implements research substrate gate item #16; no new ADR required

## Objective

Define a reproducible experiment-bundle abstraction. An experiment bundle groups a label configuration, a split configuration, factor definitions, and metadata into a single deterministic unit with a content fingerprint. The fingerprint is stable — same inputs produce the same fingerprint — enabling preregistration and exact reproduction.

## Required Contract

- **ExperimentBundle** immutable value object with:
  - `label_config` (LabelConfig)
  - `split_config` (SplitConfig)
  - `factor_defs` (list of factor identifiers; factor materialization out of scope)
  - `metadata` (frozen dict: name, description, creation_time, etc.)
  - `fingerprint` (deterministic SHA-256 of canonical serialization)
- **ExperimentBundleRegistry** — validates and persists bundles:
  - `register(bundle)` → verifies fingerprint, rejects duplicates
  - `load(fingerprint)` → returns bundle or fail-closed
  - `list_bundles()` → all registered fingerprints
- Deterministic canonical serialization to bytes for fingerprint computation.
- Fail-closed: corrupt/missing bundles → error, no silent fallback.
- Factor identifiers are opaque strings (factor materialization out of scope).
