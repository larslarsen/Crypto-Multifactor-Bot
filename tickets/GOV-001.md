# GOV-001 — Establish repository governance artifacts and agent controls

**Status:** ACCEPTED

**Priority:** P0  
**Dependencies:** CAT-001A complete, architecture freeze (ADR-0007)  
**Layer:** governance / meta

## Objective

Introduce the minimal set of governance files and processes so that future work (including by AI agents) is traceable, reviewable, and self-documenting.

## Deliverables

- Root `AGENTS.md` with agent operating rules.
- `tickets/GOV-001.md` (this ticket).
- ADR 0011 documenting the governance model.
- `scripts/check_repo_control.py` — basic artifact presence and structure checker.
- Update `docs/handoff/CURRENT_TASK.md` to point to GOV-001.
- Update README status section to reflect governance milestone.

## Invariants

- All future tickets must be traceable.
- Architecture decisions must go through ADRs.
- Agents must be given explicit instructions via AGENTS.md.
- The control script must pass on main before review handoff.
- No work proceeds until the current task ticket is present for senior review.

## Required Checks

Run before review:

```bash
python scripts/check_repo_control.py
```

## Acceptance

- All deliverables above exist on the local `main` branch.
- `scripts/check_repo_control.py` passes.
- The owner publishes the commit; the senior engineer reviews the work directly from the repository.
