# RAW-001 — Content-addressed raw object writer

**Priority:** P0  \
**Dependencies:** CAT-001 (satisfied via CAT-001A, `ACCEPTED`)  \
**Layer:** ingest + catalog
**Status:** AWAITING_REVIEW

## Objective

Persist exact acquired bytes immutably and register them in the catalog.

## Deliverables

- streaming writer that does not load the object into RAM;
- SHA-256 and byte count during write;
- same-filesystem temporary file, flush/fsync, atomic rename;
- deterministic path such as `raw/sha256/ab/cd/<hash>`;
- source/request/response metadata registration;
- idempotent duplicate handling;
- collision and corruption checks;
- orphan-temp reconciliation command.

## Invariants

- Existing content is never overwritten.
- A catalog row never points to partial bytes.
- Same bytes map to the same content object.
- Source identity and acquisition metadata remain separate from byte identity.
- Failed acquisition creates a failed run record but no accepted raw object.

## Required tests

Include empty object, multi-chunk object, duplicate bytes, interrupted write, preexisting corrupt destination, retry, and concurrent duplicate attempt.


## Agent stop condition

After the acceptance commands pass, produce a change report and stop. Do not begin the next ticket.
