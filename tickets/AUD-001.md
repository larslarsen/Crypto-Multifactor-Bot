# AUD-001 — Schema and coverage profiler

**Priority:** P0  
**Dependencies:** LEG-001, MAN-001  
**Layer:** quality  
**Status:** IN_PROGRESS

## Objective

Profile candidate files into machine-readable reports before source-specific normalization.

## Deliverables

- bounded sampling and streaming full-pass modes;
- inferred physical schema and explicit uncertainty;
- row counts, timestamp min/max, nulls, duplicates, monotonicity, frequency/gaps;
- numeric ranges and impossible OHLC checks where columns map clearly;
- report Parquet/JSON plus manifest;
- quality issues, not silent repairs.


## Reviewer verdict

- **Review target:** commit `9e1a0fc3e0a5b08b14cee23c1d2dc6a9ab8fcf3d`
  (`feat(aud-001): integrate schema & coverage profiler + focused tests`).
- **Verdict:** `CHANGES_REQUIRED` — the implementation is structurally sound and the
  acceptance commands pass, but four reviewer findings must be resolved before
  acceptance:
  1. **Bounded FULL-mode processing:** the streaming full-pass (`ProfileMode.FULL`)
     must stay memory-bounded; confirm the implementation does not materialize the
     entire candidate in RAM and document the bound.
  2. **Valid MAN-001 statistics:** profiler-produced artifacts must carry valid
     MAN-001 statistics (correct byte sizes, SHA-256, row counts) so the staged
     `PublishPlan` is internally consistent and verifiable.
  3. **SHA-256 verification:** the profiler must verify (not merely record) the
     candidate `content_sha256` against the file bytes, or explicitly document why
     verification is deferred to the caller.
  4. **Preserved Parquet physical types:** the Parquet detail/issues artifacts must
     preserve physical column types (no silent coercion that loses precision or
     schema information).
- A Sr Dev correction drop is expected; Hermes will integrate and validate it under
  the current authorized scope. Ticket remains `IN_PROGRESS`; next ticket `NONE`.
