## Objective

Build ML-based factors that learn from historical market data to predict
cross-sectional returns. These challengers compete against the transparent
baselines (BASE-001) and simple composites (COMP-001), testing whether
learning adds value over fixed formulas. Per preregistration, ML factors
must beat simple composites net of costs.

## Required Contract

- Each factor implements `Factor` protocol from `cryptofactors.factors.contract`
- Deterministic given universe + as_of (fixed random seed)
- Training window respects as-of boundary (no lookahead). At `decision_time T`, the training set consists of past observation times `t` such that the label interval `[t + min_gap, t + min_gap + horizon]` ended on or before `T`.
- Expanding window: the model trains on all valid historical cross-sections up to `T`.
- The factor computes the 3 baseline features (momentum, mean_reversion, volume) internally or accepts a feature store.
- Returns `FactorFrame` containing `score` (model prediction) for the current `decision_time T`.

## ML Factors (in order)

1. **Ridge Regression** — L2-regularized linear model on baseline factor features. 
   - `sklearn.linear_model.Ridge`
   - Hyperparameter grid: `alpha in {0.1, 1.0, 10.0}`.
   - Features are standardized before fitting.

2. **Elastic Net** — L1+L2 regularized linear model. 
   - `sklearn.linear_model.ElasticNet`
   - Hyperparameters: `alpha in {0.1, 1.0}`, `l1_ratio in {0.3, 0.7}`.
   - Features are standardized before fitting.

3. **Shallow Gradient-Boosted Tree** — XGBoost with limited depth.
   - `xgboost.XGBRegressor`
   - Hyperparameters: `max_depth in {2, 3}`, `n_estimators in {50, 100}`, `learning_rate in {0.05, 0.1}`.
   - Fixed seed (e.g., `random_state=42`).

## Feature Strategy

- Features are the raw values of the three baseline factors (momentum, mean-reversion, volume).
- Target is the `FORWARD_RETURN` label generated by `LabelEngine`.
- At `decision_time T`, training labels are bounded by `event_end <= T`.
- Prediction is made on features at `T`.

## Test Strategy

- Unit tests: deterministic with fixed seed, shape correctness, verify no lookahead bias in training dates.
- Integration: run each ML factor through research substrate (same as BASE-001/COMP-001), verify FactorValue output conforms to contract.
- Reproducibility: same inputs produce identical scores across runs.
- No performance/edge test — baselines and composites serve as performance comparators.
